# OpenJCK Protocol v1 Specification

**Version:** 1.0  
**Status:** Stable  
**Date:** April 2026  

---

## Overview

The OpenJCK Protocol is an open, language-agnostic standard for emitting agent observability events. It enables any agent framework—written in any language—to send telemetry to OpenJCK Cloud or any compatible receiver.

Like OpenTelemetry for distributed tracing, the OpenJCK Protocol is designed for autonomous AI agent behavior: session lifecycle, token usage, tool calls, flags, and errors.

### Principles

- **Zero dependency**: Emitters require only HTTP and JSON.
- **Unauthenticated in v1**: Open standard; no API keys or OAuth. (Rate limiting applies.)
- **Idempotent**: Duplicate events are harmless; servers may deduplicate.
- **Extensible**: New event types may be added in minor versions; major versions break compatibility.

---

## Transport

- **Endpoint**: `POST /api/protocol/events`
- **Content-Type**: `application/json`
- **Authentication**: None (v1). Protocol is open.
- **Rate Limiting**: Receivers SHOULD apply reasonable rate limits (e.g., 1000 req/min per IP).
- **Request Body**: A single event object OR an array of event objects.

**Response:**
```json
{
  "accepted": 3,
  "rejected": 1,
  "errors": [
    {
      "event": { /* sanitized event with masked IDs */ },
      "reason": "Missing required field: org_id"
    }
  ]
}
```

---

## Event Envelope (Required Fields)

All events MUST include these top-level fields:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `at_version` | string | Yes | Protocol version (e.g., `"1.0"`). Major version must be ≤ 1. |
| `event` | string | Yes | Event type: `"session.start"`, `"session.step"`, `"session.flag"`, `"session.end"`. |
| `session_id` | string | Yes | Unique identifier for the agent session. UUID recommended. |
| `org_id` | string | Yes | Organization identifier (UUID). Used to route events to the correct dashboard. |
| `timestamp` | string | Yes | ISO 8601 timestamp (e.g., `"2025-04-12T12:34:56.789Z"`). |

## Event Types

### 1. session.start

Signals the beginning of an agent session.

**Additional fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `claw_name` | string | No (default: `"unknown"`) | Human-readable agent identifier. |
| `project` | string | No | Optional project name. |
| `environment` | string | No (default: `"dev"`) | Environment label (`"dev"`, `"staging"`, `"prod"`). |
| `tags` | array[string] | No (default: `[]`) | User-defined tags. |
| `metadata` | object | No (default: `{}`) | Arbitrary key-value pairs. |
| `guard_config` | object | No | Guard configuration used for this session. |

**Example:**
```json
{
  "at_version": "1.0",
  "event": "session.start",
  "session_id": "sess-abc123",
  "org_id": "org-xyz789",
  "timestamp": "2025-04-12T12:00:00.000Z",
  "claw_name": "code-writer",
  "project": "backend-refactor",
  "environment": "prod",
  "tags": ["team:api", "priority:high"],
  "metadata": { "git_branch": "main" }
}
```

---

### 2. session.step

Emitted after each LLM API call or agent step.

**Additional fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `step` | integer | Yes | Sequential step number (0-indexed). |
| `usage` | object | Yes | Token usage metrics. Contains: `input_tokens`, `output_tokens`, `tool_call_count`, `step_number` (optional, overrides `step`). |
| `request` | object | Yes | Request context. Contains: `model` (string). |
| `step_packet` | object | No | **Full step packet** as defined by OpenJCK (recommended). Includes tools, guard, error, and recording fields. This allows full replay. |

**Example (with step_packet):**
```json
{
  "at_version": "1.0",
  "event": "session.step",
  "session_id": "sess-abc123",
  "org_id": "org-xyz789",
  "timestamp": "2025-04-12T12:00:05.000Z",
  "step": 0,
  "usage": {
    "input_tokens": 1842,
    "output_tokens": 341,
    "tool_call_count": 1,
    "step_number": 0
  },
  "request": {
    "model": "claude-sonnet-4"
  },
  "step_packet": {
    "schema_version": "1.0",
    "sdk": { "name": "openjck", "version": "0.3.0", "language": "python" },
    "session": { /* ... */ },
    "event": { "event_type": "step_end", "step_number": 0 },
    "request": { "model": "claude-sonnet-4" },
    "usage": { /* ... */ },
    "tools": [ /* ... */ ],
    "guard": { "triggered": false },
    "error": null,
    "recording": { "record": true }
  }
}
```

---

### 3. session.flag

Signals that a guard rule or anomaly was triggered.

**Additional fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `flag_type` | string | Yes | Type of flag (e.g., `"loop_detected"`, `"cost_limit"`, `"tool_quota"`). |
| `detail` | string | No | Human-readable description (recommended). |

**Example:**
```json
{
  "at_version": "1.0",
  "event": "session.flag",
  "session_id": "sess-abc123",
  "org_id": "org-xyz789",
  "timestamp": "2025-04-12T12:05:00.000Z",
  "flag_type": "loop_detected",
  "detail": "read_file called 3 times with identical path 'src/main.py' in 10-step window"
}
```

---

### 4. session.end

Marks the termination of a session.

**Additional fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `status` | string | No (default: `"completed"`) | Final status: `"completed"`, `"failed"`, `"terminated"`. |
| `failure_root_cause` | string | No | If status is `failed` or `terminated`, the root cause description. |
| `ended_at` | string | No | Timestamp when the session ended (defaults to event `timestamp`). |
| `summary` | object | No | Cumulative metrics: `steps`, `input_tokens`, `output_tokens`, `cost_usd`. |

**Example:**
```json
{
  "at_version": "1.0",
  "event": "session.end",
  "session_id": "sess-abc123",
  "org_id": "org-xyz789",
  "timestamp": "2025-04-12T12:10:00.000Z",
  "status": "completed",
  "ended_at": "2025-04-12T12:10:00.000Z",
  "summary": {
    "steps": 12,
    "input_tokens": 15000,
    "output_tokens": 4500,
    "cost_usd": 0.1234
  }
}
```

## Idempotency

Servers SHOULD handle duplicate events gracefully. In particular:

- `session.start`: Duplicate `session_id` and `event="session.start"` → ignore or return 200 (no new session created).
- `session.step`: Duplicate `(session_id, step_number)` → ignore (no double-counting). The `step_packet` may be updated (upsert).
- `session.end`: Duplicate `session_id` and `event="session.end"` → ignore or ensure idempotent update.

The receiver response includes `accepted` (new events processed) and `rejected` (errors). A duplicate that is silently ignored counts as **accepted** (HTTP 200) but does not change state.

---

## Versioning

- Protocol version expressed as `major.minor` (semantic versioning).
- **Major version changes** (e.g., 2.0) may break compatibility. Receivers MUST reject `at_version` with major > 1 in v1 servers.
- **Minor version changes** (e.g., 1.1) are backward-compatible: new optional fields may be added, but existing fields remain unchanged.
- Emitters SHOULD set `at_version` to the version they implement. Receivers may ignore unknown minor versions if major matches.

---

## Error Responses

The receiver returns HTTP 200 with a JSON body summarizing results:

```json
{
  "accepted": 2,
  "rejected": 1,
  "errors": [
    {
      "event": { "session_id": "sess-abc...", "org_id": "org-xyz...", "event": "session.invalid" },
      "reason": "Unknown event type: session.invalid"
    }
  ]
}
```

Errors include a sanitized version of the event (with long IDs truncated) to aid debugging.

## Python Usage

The OpenJCK Python SDK includes a `ProtocolEmitter` class that requires **zero dependencies** beyond the standard library.

```python
from openjck import ProtocolEmitter

# Initialize
emitter = ProtocolEmitter("http://localhost:7070/api/protocol/events")

# Start a session
emitter.session_start(
    org_id="org-123",
    session_id="sess-456",
    claw_name="researcher",
    environment="prod"
)

# Emit a step (include step_packet for full replay)
emitter.session_step(
    org_id="org-123",
    session_id="sess-456",
    step=0,
    model="claude-sonnet-4",
    input_tokens=1842,
    output_tokens=341,
    tool_call_count=1,
    step_packet={ /* full step packet JSON as recorded by ClawSession */ }
)

# Flag a loop
emitter.session_flag(
    org_id="org-123",
    session_id="sess-456",
    flag_type="loop_detected",
    detail="read_file called 3x with same path"
)

# End the session
emitter.session_end(
    org_id="org-123",
    session_id="sess-456",
    status="completed",
    summary={
        "steps": 10,
        "input_tokens": 15000,
        "output_tokens": 4500,
        "cost_usd": 0.12
    }
)
```

### Raw emit

For custom events or future extensions:

```python
emitter.emit({
    "event": "custom.type",
    "session_id": "sess-456",
    "org_id": "org-123",
    "timestamp": "2025-04-12T12:00:00Z",
    "custom_field": "value"
})
```

## Implementation Notes

### Server Behavior

- The receiver processes events **independently** and returns aggregate results.
- On database constraint violations (e.g., duplicate step), the server should count the event as **accepted** (no error) if the idempotency rule applies.
- Errors due to validation, missing required fields, or unknown event types count as **rejected**.
- The `errors` array is capped (e.g., 100 entries) to avoid huge responses.

### org_id Assignment

In v1, `org_id` is supplied by the emitter. There is no authentication; the protocol relies on the emitter to provide the correct org_id. In future versions, this may be replaced by scoped API keys or signed tokens.

### Step Packet Schema

The `step_packet` field is optional but **strongly recommended**. If provided, it should match the OpenJCK Step Packet schema (see Master Truth). This enables full replay in the dashboard.
